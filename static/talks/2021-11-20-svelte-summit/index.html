You can build your Svelte components in a separate folder

In the default rollup.config.js:

beforeafter
input: 'src/main.js',
output: {
	sourcemap: true,
	format: 'iife',
	name: 'app',
	file: 'public/build/bundle.js'
},

input: 'src/App.svelte',
output: {
	sourcemap: true,
	format: 'umd',
	name: 'MySvelteComponent',
	file: 'public/build/bundle.js'
},

What if you're already using a JavaScript framework?

React? Vue? Angular???

In the default package.json:

beforeafter
{
	"name": "svelte-app",
	"version": "1.0.0",
	

{
	"name": "my-svelte-component",
	"main": "public/build/bundle.js",
	"version": "1.0.0",

You can install your Svelte component from a local folder

📁

npm install ./my-svelte-component

Or you can install from a Git repo:

📁

npm install git+https://my.git.server/my-svelte-component.git

Import your Svelte component like any other node module

const MySvelteComponent = require('my-svelte-component');
require('my-svelte-component/public/build/bundle.css');

or

import MySvelteComponent from 'my-svelte-component';
import 'my-svelte-component/public/build/bundle.css';

Use the same instantiation code

new MySvelteComponent({
	target: root,
	props: {
		// props here
	}
});

Two build processes too much?

Webpack?

svelte-loader!

module: {
	rules: [
		{
			test: /\.(html|svelte)$/,
			use: 'svelte-loader'
		},
	]
}

Rollup?

rollup-plugin-svelte!

import svelte from 'rollup-plugin-svelte';

...

plugins: [
	svelte()
]
